# Руководство по написанию Markdown для постов (Fidd)

Это краткое руководство по использованию Markdown в постах для корректного отображения в нашем SPA компоненте `PostPage`.

## Базовое форматирование
Поддерживается весь стандартный синтаксис:

### Заголовки
Согласно спецификации, поддерживаются все 6 уровней заголовков:
```markdown
# Заголовок уровня 1
## Заголовок уровня 2
### Заголовок уровня 3
#### Заголовок уровня 4
##### Заголовок уровня 5
###### Заголовок уровня 6
```

### Текст и ссылки
* **Жирный текст**: `**текст**`
* *Курсив*: `*текст*`
* [Ссылки](https://example.com)

## Таблицы
Поддерживаются классические таблицы (GitHub Flavored Markdown):

| Заголовок 1 | Заголовок 2 |
|-------------|-------------|
| Ячейка 1    | Ячейка 2    |
| Ячейка 3    | Ячейка 4    |

## Медиафайлы (Изображения, Видео, Аудио)

Так как все файлы загружаются локально в `.fidd` архив (или лежат рядом с `post.md`), используйте **относительные пути**. Наш рендерер на странице поста автоматически подставит нужный URL к бэкенду.

### Изображения
Для вставки картинки используйте стандартный синтаксис:
```markdown
![Описание картинки](image.png)
![Схема обмена](folder/schema.jpg)
```

### Видео
Для видео используйте тот же синтаксис картинок, но с расширением видео-файла (например, `.mp4`, `.webm`, `.mov`). Наш плеер сам поймет, что это видео:
```markdown
![Демонстрация работы](demo.mp4)
```

### Аудио
Аналогично для аудио (`.mp3`, `.wav`, `.ogg`):
```markdown
![Подкаст или трек](audio.mp3)
```

> **Техническая деталь:** При рендеринге относительные пути `image.png` перехватываются компонентом `<Markdown>` и заменяются на `http://localhost:4198/fidds/v1/{fiddId}/{messageNumber}/image.png`.
